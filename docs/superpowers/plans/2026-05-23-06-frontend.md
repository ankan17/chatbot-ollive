## State management choice (justification — read before Task 1)

**Decision: React built-ins only — Context for cross-cutting read-mostly state (session), `useReducer` for the two genuinely stateful machines (chat send/cancel, guest trial), local `useState` everywhere else. No Redux, no Zustand.**

Rationale: the app has exactly two pieces of non-trivial mutable state — the **active chat stream** (idle → sending → streaming → done/error/cancelled, with a live-appending buffer) and the **guest trial** (a small state machine mirrored to `localStorage`). Both are naturally a reducer + a few actions; a reducer is independently testable as a pure function (which is exactly our testing posture). Everything else is server state fetched per-view (conversation list, a conversation's history, metrics) — owned by the container that needs it via a fetch hook, not a global store. Session/auth status is the one truly cross-cutting value, so it lives in a `SessionContext`. Pulling in Redux/Zustand would add a dependency and ceremony for state that two reducers and one context already model cleanly (DRY/YAGNI). If server-cache invalidation later becomes painful, TanStack Query is the documented upgrade point — but it is not warranted at this scope.

  ```ts
  export const API_BASE_URL: string; // import.meta.env.VITE_API_BASE_URL
  export function buildUrl(path: string, query?: Record<string, string | number | undefined>): string;

  // errors.ts — the contract's AppErrorCode (imported) + one client-only code.
  // AppErrorCode (from @ollive/shared/api) = 'validation_error' | 'unauthorized' | 'login_required'
  //   | 'not_found' | 'rate_limited' | 'internal_error'. We do NOT redefine it.

    readonly status: number;       // 0 for network_error
    readonly details?: unknown;    // e.g. validation_error field details (ApiErrorBody.details)
  ```

- **Algorithm:**
  - `request<T>()`: build URL via `buildUrl`; call `fetch(url, { method, credentials: 'include', headers: body ? {'content-type':'application/json'} : undefined, body: body && JSON.stringify(body), signal })`. On a thrown fetch (network/offline) → throw `ApiError('network_error', 0, ...)` (but **re-throw `AbortError` untouched** so stream/cancel callers can detect intentional aborts — see Task 5). On `204` → resolve `undefined as T`. Read body as JSON when present (guard for empty). If `!res.ok` → `throw normalizeError(res.status, parsedBody)`. Else return parsed body as `T`.
  - `normalizeError(status, body)`: read `body.error` (the contract's `AppErrorCode`) when present. Map per the API-CONTRACTS.md §7 error catalog: `400`→`validation_error` (carry `body.details` from `ApiErrorBody`); `401`→`unauthorized`; `403` + `body.error === 'login_required'`→`login_required` (also capture `body.remaining` from `LoginRequiredBody` onto `ApiError.remaining` for the guest-cap UI); `404`→`not_found`; `429`→`rate_limited`; `>=500`→`internal_error`; otherwise prefer a recognized `body.error` code, else `internal_error` fallback. Message = `body.message ?? body.error ?? statusText`. (The frontend-only `network_error` is produced by `request<T>()` on a thrown fetch, not by `normalizeError`.)
  - `buildUrl`: prefix `API_BASE_URL`, append `path`, serialize defined query params (drop `undefined`) via `URLSearchParams`.
  - Endpoint wrappers are one-liners over `request<T>()` with the right path/method/query and the DTO type parameter. `googleSignInUrl()` returns a string (the app does a full-page navigation to it — OAuth redirect, not a fetch).
- **Patterns / decisions / edge cases:** API-client-module pattern (single seam for all network I/O → trivially mockable in tests via `vi.mock('../api/...')`). `credentials:'include'` on every call so the httpOnly session/guest cookies ride along across origins (CORS `WEB_ORIGIN` in dev). The client never reads or stores tokens. `getSession` is the only call that "never 401s" by contract — UI uses it for the guest indicator. Edge cases: `204` no-content (logout); empty/non-JSON error bodies (fall back to statusText); AbortError must pass through unwrapped.

**Test cases (write first where applicable):** none here; `normalizeError`'s 401/403 branches are asserted via Task 4's `useSession` test against a mocked `getSession`/`getMe`. (If the implementer prefers, a 2-3 case pure unit test of `normalizeError` is welcome but optional.)

**Design:**
- **Signatures / types:**
  ```ts
  // SSE event payloads are the CONTRACT types — imported, not redefined (API-CONTRACTS.md §3 / §8 chat.ts).
  //   SseStartData = { messageId: string | null; requestId: string }   (messageId null for guest chat)
  //   SseTokenData = { delta: string }
  //   SseDoneData  = { messageId: string | null; finishReason: string; usage: Usage }   (usage ALWAYS present)
  //   SseErrorData = { code: 'rate_limited'|'provider_timeout'|'provider_error'|'internal_error'; message: string }
  //   SseEvent     = discriminated union over { event, data } for the four event names.
  import type { SseEvent, SseStartData, SseTokenData, SseDoneData, SseErrorData } from '@ollive/shared/api';

  // Pure parser: feed it raw decoded text chunks, get back complete events.
  // Holds a buffer between calls; emits only fully-terminated frames (blank-line delimited).
  export interface SseFrameParser {
    push(textChunk: string): SseEvent[]; // returns events completed by this chunk
    flush(): SseEvent[];                 // events from any trailing buffered frame at stream end
  }
  export function createSseParser(): SseFrameParser;

  export interface StreamChatCallbacks {
    onStart?(d: SseStartData): void;
    onToken?(d: SseTokenData): void;
    onDone?(d: SseDoneData): void;
    onError?(d: SseErrorData): void;   // a server-sent `event: error` frame (NOT a thrown JS error)
  }
  export interface StreamChatOptions extends StreamChatCallbacks { signal?: AbortSignal; }

  // POSTs body to url, reads the SSE stream, dispatches callbacks. Resolves when the
  // stream ends (after `done`/`error` or EOF). Rejects on network failure or non-OK
  // pre-stream status (e.g. 403 login_required surfaces as ApiError). Aborting the
  // signal stops the read and rejects with the underlying AbortError.
  export function streamChat(url: string, body: unknown, opts: StreamChatOptions): Promise<void>;
  ```
- **Algorithm:**
  - **`createSseParser`** — keep a string `buffer`. `push(chunk)`: append to buffer; split into complete frames on the blank-line delimiter (`\n\n`, tolerate `\r\n\r\n`), leaving any incomplete trailing frame in the buffer; parse each complete frame → `{ event, data }`. Frame parse: read `event:` line(s) (default `message` if absent, but our server always sends one) and concatenate `data:` line value(s); `JSON.parse` the joined data; narrow to the typed `SseEvent` by `event` name; skip comment lines (`:` prefix — heartbeats, ST5) and unknown event names. `flush()`: if a trailing non-empty frame remains (stream ended without final blank line), parse and return it; else `[]`.
  - **`streamChat`** — `fetch(url, { method: 'POST', credentials: 'include', headers: { 'content-type':'application/json', accept:'text/event-stream' }, body: JSON.stringify(body), signal })`. If `!res.ok` (e.g. guest 403) → parse JSON body and `throw normalizeError(res.status, body)` *before* streaming. Else get `reader = res.body!.getReader()`, a `TextDecoder`, and a parser; loop `reader.read()`; decode each chunk (`{ stream: true }`), `parser.push(text)`, and for each event dispatch the matching callback. On `done`/EOF, call `parser.flush()` and dispatch any final event, then resolve. If the read loop throws because the signal aborted (`err.name === 'AbortError'` or `signal.aborted`) → reject with that AbortError (caller treats as cancel). Other thrown errors → wrap as `ApiError('network_error', 0, ...)` and reject.
- **Patterns / decisions / edge cases:** SSE-stream-parser pattern decoupled from React (pure + framework-free → fully unit-testable, reused by both authed and guest chat). Callback-based (not async-iterator) because callbacks compose directly with the chat reducer's dispatch and keep cancel handling simple. **Per the contract (§3), the event sequence is `start` → `token`* → (`done` | `error`); `done.usage` is ALWAYS present (typed `Usage`, never null); and cancel emits NO `done`/`error` — the stream simply closes (the client detects its own `AbortError`).** Edge cases the tests pin: a single event split across two network chunks (parser buffers and completes it); multiple events in one chunk; a heartbeat comment line (`: ping`) interleaved (ignored, ST5); a `done` frame carrying a populated `usage` object; abort mid-stream closing the stream with no `done`/`error` and rejecting cleanly; a 403 `login_required` thrown before any token.

**Test cases (write first, TDD):** drive `streamChat` with a hand-built `Response` whose `body` is a `ReadableStream` of `Uint8Array` chunks (mock `globalThis.fetch` with `vi.fn`).
- Parser — single complete frame: push `event: token\ndata: {"delta":"Hi"}\n\n` → one `{event:'token', data:{delta:'Hi'}}`.
- Parser — frame split across pushes: push `event: token\ndata: {"de`, then `lta":"Hi"}\n\n` → first push returns `[]`, second returns the completed token event.
- Parser — multiple frames in one push: a `start` then two `token` frames concatenated → three events in order.
- Parser — ignores heartbeat/comment lines (`: ping\n\n`) and unknown event names (no events emitted, buffer stays clean).
- Parser — `flush()` returns a trailing frame that ended without a final blank line.
- `streamChat` happy path: stream yields `start` → `token`×3 → `done` (carrying a populated `usage: Usage`) → `onStart`, `onToken`×3 (deltas in order), `onDone` (with a non-null `usage`) fire once each; promise resolves.
- `streamChat` server error event: stream yields `start` → `error` (`{code:'rate_limited',message:'...'}`) → `onError` fires, promise resolves (server-sent error is not a thrown rejection).
- `streamChat` cancel: abort the signal after the first token → read loop stops, the stream closes with NO `done`/`error` callback, `onToken` fired at most for already-received frames, promise rejects with an AbortError.
- `streamChat` guest cap: `fetch` resolves a `403 { error:'login_required', remaining:0 }` → promise rejects with `ApiError` code `login_required`, no callbacks fired.

- **Algorithm:**
  - `chatReducer` (pure): `sendUser` → push an optimistic user `ChatMessage` (`status:'complete'`), `phase:'sending'`. `streamStart` → push an empty assistant message with `data.messageId`, set `currentAssistantId`, `phase:'streaming'`. `token` → append `delta` to the message matching `currentAssistantId` (immutable map). `streamDone` → set that assistant message `status:'complete'` and `tokenCount = data.usage.completionTokens` (the contract guarantees `done.usage` is present — `Usage`, never null), `phase:'done'`, clear `currentAssistantId`. `streamError` → set assistant `status:'error'`, store `error`, `phase:'error'`. `cancelled` → set assistant `status:'partial'`, `phase:'cancelled'` (no `done` arrives on cancel, so the partial content is whatever streamed). `reset` → replace `messages`, `phase:'idle'`.

- **Patterns / decisions / edge cases:** reducer pattern keeps the streaming state machine pure and unit-testable; the hook owns only the imperative glue (fetch + AbortController). Optimistic user message uses a temp id then is reconciled implicitly (server doesn't echo the user message id over SSE; on next full resume the canonical ids load). Edge cases pinned by tests: a `token` arriving with no current assistant id (defensive no-op); cancel after partial output keeps the partial text and marks `partial` (no `done`/`error` arrives on cancel, per §3); `streamDone` always carries a `usage` object, so the assistant message's `tokenCount` is set from `usage.completionTokens`.

- `streamStart` appends an empty assistant message with the server `messageId` and sets `phase:'streaming'` + `currentAssistantId`.
- `token` ×3 appends deltas in order to the current assistant message, accumulating its content.
- `streamDone` marks the assistant message `complete`, sets `phase:'done'`, clears `currentAssistantId`, and sets `tokenCount` from the (always-present) `usage.completionTokens`.
- `streamError` marks the assistant message `error`, stores the error, `phase:'error'`.
- `cancelled` after some tokens keeps the partial content and sets that message `status:'partial'`, `phase:'cancelled'`.
- `token` with no `currentAssistantId` is a no-op (state unchanged).
- `reset` replaces messages and returns `phase:'idle'`.

**Design:**
- **Signatures / types:**

  ```ts
  // ChatMessage is the alias for the contract `Message` DTO (@ollive/shared/api); GuestMessageInput is the client-local import shape.
  import type { GuestMessageInput, ChatMessage } from '../api/types';

  // Guest conversation held entirely client-side (PRD §7.1, mirrored to localStorage).
  export interface GuestConversation {
    clientConversationId: string;          // crypto.randomUUID() — idempotency key for import (API-CONTRACTS.md §2 import)
    messages: ChatMessage[];               // local-only ids/sequence; role user|assistant
    createdAt: string;
  }
  export type GuestPhase =
    | 'idle'        // no message sent yet
    | 'sending' | 'streaming'
    | 'awaiting'    // turn complete, can send again (under cap)
    | 'capped'      // server returned 403 login_required (or local count >= limit)
    | 'error';

    conversation: GuestConversation;
    phase: GuestPhase;
    sentUserCount: number;                 // number of user turns sent this trial
    currentAssistantId?: string;
    error?: { code: string; message: string };
  }
  export type GuestAction =
    | { type: 'sendUser'; content: string; id: string }
    | { type: 'streamStart'; assistantId: string }
    | { type: 'token'; delta: string }
    | { type: 'streamDone' }
    | { type: 'streamError'; code: string; message: string }
    | { type: 'cancelled' }
    | { type: 'capped' }                                  // 403 login_required
    | { type: 'hydrate'; state: GuestState };             // restore from localStorage

  export function createInitialGuestState(): GuestState;  // fresh conversation + clientConversationId

  // localStorage codec
  export const GUEST_STORAGE_KEY = 'ollive.guest.v1';
  export function loadGuestState(): GuestState | undefined;     // parse + validate; undefined if absent/corrupt
  export function saveGuestState(state: GuestState): void;      // JSON.stringify under GUEST_STORAGE_KEY
  export function clearGuestState(): void;

  // import mapping: GuestConversation -> POST /v1/conversations/import body
  export function toImportRequest(c: GuestConversation): { clientConversationId: string; messages: GuestMessageInput[] };
  // maps each message to { role: 'user'|'assistant', content }, dropping ids/sequence/timestamps,
  // preserving order; 'system' messages (none expected) are excluded.

  // useGuestChat.ts
  export interface UseGuestChat {
    state: GuestState;
    remaining: number;               // from session guest.remaining (server truth), floored at 0
    limit: number;
    isStreaming: boolean;
    isCapped: boolean;               // phase === 'capped'
    send(content: string): void;     // POST /v1/guest/messages with full local history
    stop(): void;
    importOnLogin(): Promise<import('../api/types').ConversationDetail>; // after OAuth landing on `/` flips session to authed
  }
  export function useGuestChat(): UseGuestChat;
  ```
- **Algorithm:**
  - `guestReducer` mirrors the chat reducer but adds the trial bookkeeping: `sendUser` pushes a user message, increments `sentUserCount`, `phase:'sending'`. `streamStart`/`token`/`streamDone`/`streamError`/`cancelled` behave like the authed reducer over the local `conversation.messages`; `streamDone` → `phase:'awaiting'`. `capped` → `phase:'capped'` (the server said `login_required`). `hydrate` replaces the whole state from storage.
  - **State machine transitions:** `idle/awaiting --sendUser--> sending --streamStart--> streaming --streamDone--> awaiting`; any send/turn that the server rejects with 403 → `--capped--> capped`; `streaming --cancelled--> awaiting`; `* --streamError--> error` (recoverable: a subsequent `sendUser` returns to `sending`). The cap is **server-authoritative** (403 drives `capped`); the local `sentUserCount`/session `remaining` only inform the banner/optimistic disable — never the sole gate (PRD: cap enforced server-side, not client-trusted).
  - `useGuestChat`: `useReducer(guestReducer)`; on mount, `loadGuestState()` → `hydrate` if present (survives refresh — FE10), else `createInitialGuestState()`. `send(content)`: dispatch `sendUser`; build the guest body `{ messages: conversation.messages.map(toRole/content), content }` and `streamChat(buildUrl('/v1/guest/messages'), body, callbacks)`. Stream callbacks dispatch the same actions as the authed path; **on the promise rejection, if it's `ApiError` code `login_required` → dispatch `capped`**; if AbortError → `cancelled`; else → `streamError`. After each completed turn, call `useSession().refresh()` so the banner's `remaining` reflects the server counter. **Persist on every state change** (`useEffect` → `saveGuestState`). `importOnLogin()`: `importConversation(toImportRequest(state.conversation))`, then `clearGuestState()`, return the persisted conversation (the container navigates to `/c/:id` and `reset()`s the authed chat — seamless resume, §7.1 step 5-6).
  - `GuestBanner`: shows `remaining`/`limit` from session ("1 message left — sign in to continue"). `GuestSignInPrompt`: replaces the composer when `isCapped`, with a Google sign-in button.
  - `ChatView` (unauthenticated branch): renders `<GuestBanner>` + `<MessageList>` from `state.conversation.messages` + (`isCapped` ? `<GuestSignInPrompt>` : `<Composer>`). **OAuth landing (API-CONTRACTS.md §1):** the Google callback redirects the browser to `WEB_ORIGIN` `/`, where `SessionProvider` re-checks `/v1/session`; when the session flips to `authenticated:true` and a buffered guest conversation exists, `ChatView` calls `importOnLogin()` **on landing** and navigates to the new conversation. The import is keyed by the buffer's `clientConversationId`, so a re-landing/refresh returns the same conversation (idempotent, §2).
- **Patterns / decisions / edge cases:** pure reducer + pure codec + pure mapping = the logic-heavy core is fully unit-testable with no DOM (the testing-posture requirement). The cap is enforced by the **server** (403) — the client treats `remaining` as a display hint and `sentUserCount` as an optimistic disable, never the security boundary (PRD SE10/AU7). `clientConversationId` is the import idempotency key (re-import of the same `(user, clientConversationId)` returns the existing conversation — API-CONTRACTS.md §2 RESOLUTION 7). Edge cases pinned by tests: corrupt/absent `localStorage` → fresh state; refresh mid-trial restores messages + count; reaching the cap mid-stream (403 before any token) → `capped` with no assistant bubble; import mapping strips local ids and preserves order; a `cancelled` guest turn still counts as a sent user turn (server counted it) so the banner reflects server `remaining`.

**Test cases (write first, TDD):** pure machine + codec + mapping (no DOM); for the hook, the live happy path is the Task 11 smoke.
- `createInitialGuestState`: fresh `clientConversationId` (uuid), empty messages, `phase:'idle'`, `sentUserCount:0`.
- `sendUser` → user message appended, `sentUserCount` incremented, `phase:'sending'`.
- `streamStart` → empty assistant message + `currentAssistantId`, `phase:'streaming'`.
- `token` ×N → deltas appended in order to the current assistant message.
- `streamDone` → assistant `complete`, `phase:'awaiting'`.
- `capped` → `phase:'capped'` (no assistant message added when capped before `streamStart`).
- `cancelled` → assistant `partial`, `phase:'awaiting'`, `sentUserCount` unchanged.
- `hydrate` → state fully replaced from a prior snapshot.
- localStorage round-trip: `saveGuestState(s)` then `loadGuestState()` deep-equals `s`; corrupt JSON under the key → `loadGuestState()` returns `undefined`; `clearGuestState()` removes the key.
- `toImportRequest`: a 1-user+1-assistant conversation → `{ clientConversationId, messages:[{role:'user',content},{role:'assistant',content}] }` in order, with no `id`/`sequence`/`createdAt` fields leaking.

### Contracts: see `docs/API-CONTRACTS.md` (authoritative — no remaining ambiguities)

Every HTTP + SSE shape this SPA consumes is pinned in **`docs/API-CONTRACTS.md`** and imported as a compiler-enforced DTO from **`@ollive/shared/api`** — the client redefines nothing. The four shapes the earlier draft had to *assume* are now fixed by the contract (and applied throughout this plan):
1. **Conversation list pagination** — `ConversationListPage = { items: ConversationSummary[]; nextCursor: string | null }`. `nextCursor` is **always present** (`string | null`), not optional; `null` terminates paging (§2 RESOLUTION 1).
2. **Series point fields** — throughput is `{ t, count }` (no `perMin`); errors is `{ t, count, errorCount, errorRate }`; tokens is `{ t, promptTokens, completionTokens, totalTokens }`; latency is `{ t, p50, p95, p99, count }`; every series response is the envelope `{ bucket, series }` (§5 RESOLUTION 3). The chart-shaping helpers consume these exact fields.
3. **`done.usage`** — **always present** on `done` (typed `Usage`, never null); cancel emits **no** `done`/`error` (the stream just closes, detected as the client's own `AbortError`), so the cancel path never relies on `done` (§3 RESOLUTION 4).
4. **OAuth landing** — the Google callback `302`-redirects to `WEB_ORIGIN` **`/`** (failure → `/?auth_error=1`); the SPA re-checks `/v1/session` on landing and runs guest import (`POST /v1/conversations/import`) when it holds a buffered guest conversation (§1 RESOLUTION 6).
