**Goal:** Build `@ollive/llm-sdk` — a standalone, storage-agnostic SDK that wraps any `LLMProvider`, captures inference metadata (latency, TTFT, token usage, status, previews), redacts PII at completion time, and ships validated `InferenceLog` payloads to a configurable ingestion URL over a buffered, non-blocking HTTP transport that never adds latency to the user-facing streamed response. The SDK reuses the on-the-wire contract (`inferenceLogSchema`) from `@ollive/shared` — it does **not** redefine it — and does **not** depend on `@ollive/db`.

**Architecture:** The SDK is a pure instrumentation layer (PRD §13). Its public surface is `withLogging(provider, config)`, a decorator that returns a new `LLMProvider` with the identical interface (SDK1). The provider abstraction (`LLMProvider`/`ChatRequest`/`StreamChunk`/`CallContext`, PRD §14) lets application code depend only on our interface, never a vendor SDK (PA1). The default adapter, `GoogleProvider`, is backed by the **Vercel AI SDK** (`ai` + `@ai-sdk/google`) and **normalizes** provider token-usage field names into our `{ promptTokens, completionTokens, totalTokens }` shape — normalizing provider differences is precisely the adapter's job (PA2). A `Redactor` scrubs PII from the fully-assembled text **then** truncation runs (redact-then-truncate, SDK10), so a PII token straddling the preview cut can never leak a fragment; redaction is fail-closed (SDK13). The transport buffers logs in a bounded in-memory queue, flushes on size/interval, retries with exponential backoff + jitter, and drops on overflow/exhaustion with a local warning (SDK4/SDK5). The whole logging path runs at completion, off the streaming hot path — chunks are yielded to the caller immediately (SDK10, NFR1).

**Tech Stack:** TypeScript 5.7, pnpm workspaces, Vitest 3 (root workspace runner), Zod 3, `@ollive/shared` (workspace), Vercel AI SDK `ai` ^5.0.0 + `@ai-sdk/google` ^2.0.0, Node 20 built-ins (`node:crypto` for `randomUUID`, `globalThis.fetch`, `node:http` for the test mock server).

**Context:** The repo root is the existing git repository at `chatbot-ollive/` (already contains `docs/PRD.md`, Plan 1's `package.json`, `pnpm-workspace.yaml`, `tsconfig.base.json`, `vitest.workspace.ts`, and the `@ollive/shared` + `@ollive/db` packages). Reference: `docs/PRD.md` §9 (Logging Schema), §13 (SDK Requirements, incl. PII redaction SDK9–SDK13), §14 (Provider Abstraction), §16 IN9 (redaction backstop relationship), §23 (Tradeoffs).

## Task 2: `PatternRedactor` — PII redaction (TDD)
**Implements:** SDK9 (pluggable `Redactor`, on by default, telemetry-only), SDK11 (pattern-based default detector), SDK12 (typed placeholders + counts); the building block for SDK10/SDK13 used in Task 4. Relates to IN9 (ingestion re-applies the same redaction as a backstop).
**Files:**
- Create: `packages/llm-sdk/src/redaction/redactor.ts` — `PatternRedactor`, `NoopRedactor`, `LlmRedactor`, `createRedactor`
- Edit: `packages/llm-sdk/src/index.ts` — add value exports
- Test: `packages/llm-sdk/test/redactor.test.ts`

**Design:**
- **Signatures / types:**
  ```ts
  import type { Redactor } from '../types';

  export class PatternRedactor implements Redactor {
    redact(text: string): { text: string; counts: Record<string, number> };
  }

  export class NoopRedactor implements Redactor {
    redact(text: string): { text: string; counts: Record<string, number> };
  }

  export class LlmRedactor implements Redactor {
    constructor(
      base?: Redactor, // defaults to a PatternRedactor
      classify?: (text: string) => { text: string; counts: Record<string, number> },
    );
    redact(text: string): { text: string; counts: Record<string, number> };
  }

  export function createRedactor(mode: 'off' | 'pattern' | 'llm', override?: Redactor): Redactor;
  // internal helpers (not exported): luhnValid(digits): boolean; shannonEntropy(s): number
  ```
- **Algorithm:**
  - **`PatternRedactor.redact(text)`** — single forward pass applying an ordered list of detectors; each detector replaces matches with its typed placeholder and increments a per-type counter. The `counts` object is initialized with all type keys at `0` so the shape is stable. Placeholders: `[EMAIL]`, `[PHONE]`, `[CREDIT_CARD]`, `[SSN]`, `[IP]`, `[IBAN]`, `[API_KEY]` (count keys: `email`, `phone`, `credit_card`, `ssn`, `ip`, `iban`, `api_key`). **Detector order matters** — run structured/specific before generic so a card or SSN is never mis-tagged as a phone or high-entropy key:
    1. **email** — local-part `@` domain `.tld`; replace the whole address (incl. local part).
    2. **iban** — 2 letters + 2 digits + 10–30 alphanumerics.
    3. **credit_card** — a digit followed by 12–18 more digits with optional single space/dash separators (13–19 digits total); **Luhn-guarded** (only replace if Luhn passes).
    4. **ssn** — US `\d{3}-\d{2}-\d{4}`.
    5. **ip** — IPv4 dotted quad, each octet 0–255.
    6. **phone** — a run starting with optional `+` and a digit, containing digits/spaces/`()`/`.`/`-`, ending in a digit; **guarded** so stripped-digit length is 7–15 (this guard keeps a 16-digit non-card run from being tagged as a phone). Runs *after* card/ssn/ip, which are already placeholders by now.
    7. **api_key (known prefixes)** — `sk-`/`pk-`/`rk-` + ≥16 alnum; `ghp_`/`gho_`/`ghu_`/`ghs_`/`ghr_` + ≥20 alnum; `AKIA` + 16 upper-alnum; `xoxb-`/`xoxa-`/`xoxp-`/`xoxr-`/`xoxs-` + ≥10.
    8. **api_key (high-entropy generic)** — token length ≥24 from `[A-Za-z0-9_-]`, **guarded** to require mixed case + a digit **and** Shannon entropy > 3.5 bits/char.
  - **`luhnValid(digits)`** — strip non-digits; reject if length <13 or >19; standard Luhn (double every second digit from the right, subtract 9 if >9, sum, `sum % 10 === 0`).
  - **`shannonEntropy(s)`** — char-frequency histogram → `-Σ p·log2(p)`; used only by the high-entropy guard.
  - **`NoopRedactor.redact(text)`** — returns `{ text, counts: {} }` unchanged.
  - **`LlmRedactor.redact(text)`** — run `base.redact` (the deterministic pattern pass) first; if no `classify` is supplied, return that result unchanged; if `classify` is supplied, run it on the already-pattern-redacted text and merge the two count maps.
  - **`createRedactor(mode, override)`** — if `override` given, return it; else `'off'`→`NoopRedactor`, `'pattern'`→`PatternRedactor`, `'llm'` *without* an override → degrade to `PatternRedactor`.
- **Patterns / decisions / edge cases:** Strategy pattern (interchangeable `Redactor` implementations) + factory (`createRedactor`). Ordered-detector pipeline (specific-before-generic) is the key correctness decision. LLM mode is **opt-in and not enabled by default** — PRD §13 SDK11 recommends running LLM redaction at the ingestion backstop (IN9), not inline; without a `classify` override it degrades to the deterministic pattern detector so privacy behavior never silently weakens. Edge cases: Luhn-failing 16-digit numbers must NOT be flagged as cards (and the phone 7–15 guard prevents them being tagged as phones either); multiple occurrences of one type each increment the count; clean text returns unchanged with all-zero counts.

**Test cases (write first, TDD):**
- Table-driven detection (one case per PII type) — each asserts the placeholder appears, the **raw value is gone**, and the type count ≥ 1, using these fixtures:
  - email: `reach me at ankan@hyperverge.co please` → `[EMAIL]`, raw `ankan@hyperverge.co` absent.
  - phone: `call +1 (415) 555-2671 tomorrow` → `[PHONE]`, raw `555-2671` absent.
  - credit_card: `card 4111 1111 1111 1111 expires soon` (Visa test number, passes Luhn) → `[CREDIT_CARD]`, raw absent.
  - ssn: `ssn 123-45-6789 on file` → `[SSN]`, raw absent.
  - ipv4: `origin 192.168.1.254 logged` → `[IP]`, raw absent.
  - iban: `pay GB82WEST12345698765432 now` → `[IBAN]`, raw absent.
  - api_key (prefix): `token sk-ABCDEFGHIJKLMNOPQRSTUVWX leaked` → `[API_KEY]`, raw absent.
- Luhn negative: `order 1234567812345678 shipped` (fails Luhn) → NOT tagged `[CREDIT_CARD]`, `counts.credit_card === 0`.
- High-entropy secret: `Authorization: Bearer aZ9kQ2wXp7Lm4Rt8Yv1Bn6Cd3Ef0Gh5Ij` → contains `[API_KEY]`, `counts.api_key ≥ 1`.
- Multiple of one type: `a@b.com and c@d.org` → `counts.email === 2`; redacted text is `[EMAIL] and [EMAIL]`.
- Clean text: `the quick brown fox jumps over the lazy dog` → unchanged, every count `0`.
- `NoopRedactor`: input `ankan@hyperverge.co` → `{ text: 'ankan@hyperverge.co', counts: {} }`.
- `createRedactor`: `'off'`→`NoopRedactor` instance; `'pattern'`→`PatternRedactor`; `'llm'` (no override)→`PatternRedactor`; `('pattern', custom)`→ returns the exact `custom` instance.

> Implementer note: for a case where only the *count* is load-bearing (e.g. the multi-email exact-string assertion), if your regex produces a slightly different but still-safe redacted string, assert the count and the "raw value gone" property and relax the exact-string match — never weaken the "raw value gone" assertion.

**Done when:** all test cases pass + `tsc --noEmit` clean; commit `feat(llm-sdk): add pattern-based PII redactor with typed placeholders`.

  // registry.ts
  import type { LLMProvider } from './types';
  export type ProviderFactory = () => LLMProvider;
  export class ProviderRegistry {
    register(name: string, factory: ProviderFactory): this; // chainable
    has(name: string): boolean;

  import type { ChatRequest, LLMProvider, StreamChunk, CallContext } from '../types';

  export function googleProviderFactory(): GoogleProvider;

- **Algorithm:**
  - **`GoogleProvider.streamChat`** (async generator):
    1. Call `streamText({ model: google(req.model), messages: req.messages, abortSignal: opts?.signal, temperature: req.temperature, maxOutputTokens: req.maxOutputTokens })`.
    2. `for await` over `result.textStream`, yielding `{ delta }` per text delta (caller sees deltas immediately).
    3. After the stream ends, `await result.usage` and `await result.finishReason`, normalize both, yield a final `{ usage, finishReason }` chunk.

  - **`ProviderRegistry`** — wraps a `Map<string, ProviderFactory>`; `register` sets and returns `this`; `create(name)` looks up the factory and calls it, throwing `No LLM provider registered for "<name>"` if absent.
- **Patterns / decisions / edge cases:** Adapter pattern (vendor SDK → our `LLMProvider`) + factory (`googleProviderFactory`) + registry (name → factory, env-configured at startup). Normalizing provider/SDK-version differences is the adapter's explicit job (PA2): absorb the `inputTokens`/`promptTokens` drift here so the rest of the system sees one shape. Adding a provider = one adapter + one registry entry; no other code changes (FR14/PA3/PA4). Edge cases: missing usage numbers coerce to `0`; `AbortSignal` is forwarded straight to the SDK as `abortSignal`. If `tsc` flags `streamText`/`textStream`/`usage`/`maxOutputTokens` mismatches, that is the version drift — reconcile against `node_modules/ai/dist/index.d.ts` while still emitting our `StreamChunk` shape.

**Files:**

**Design:**
- **Signatures / types:**
  ```ts
  import type { InferenceLog } from '@ollive/shared';
  import type { LLMProvider, InferenceLoggerConfig } from '../types';

  /** Minimal seam withLogging ships to; Task 5's BufferedHttpTransport implements it. */
  export interface LogSink {
    enqueue(log: InferenceLog): void;
  }

  /** Decorator (PRD §13 SDK1): returns an instrumented provider with the identical interface. */
  export function withLogging(
    provider: LLMProvider,
    config: InferenceLoggerConfig,
    sink: LogSink,
  ): LLMProvider;
  ```
  Test fakes to author (`test/fakes.ts`):
  - `FakeProvider implements LLMProvider` — constructed with a scripted `StreamChunk[]` and options `{ name?, throwAfter?, throwError?, delayMs?, abortAfter? }`. Its `streamChat` async-generator yields scripted chunks in order; optionally sleeps `delayMs` before the first chunk (makes TTFT measurable); throws `throwError` after `throwAfter` chunks (tests `error` status); throws an `Error` with `name === 'AbortError'` if the signal is aborted or after `abortAfter` chunks when a signal is present (tests `cancelled` status). Never calls a real model.
  - `makeMockIngestionServer({ failTimes? })` → `{ url: string; received: { auth?: string; body: unknown }[]; close(): Promise<void> }` — a `node:http` server on an ephemeral port (`listen(0)`) recording each request's `Authorization` header + parsed JSON body; returns `503` for the first `failTimes` requests, then `202`.
- **Algorithm:** Resolve `previewMaxChars = config.previewMaxChars ?? PREVIEW_MAX_CHARS` (from `@ollive/shared`) and `redactor = createRedactor(config.redaction ?? 'pattern', config.redactor)` once at decoration time. Return `{ name: provider.name, streamChat }` where `streamChat` is an async generator that, per call:
  1. Generate `requestId = randomUUID()` (`node:crypto`); record `startedAtMs = Date.now()` and ISO `startedAt`.
  2. Initialize accumulators: `ttftMs?`, `outputText = ''`, `usage?`, `finishReason?`, `status = 'success'`, `errorPayload = null`.
  3. `for await` over `provider.streamChat(req, opts)`: on a `delta` chunk, set `ttftMs = Date.now() - startedAtMs` if unset (TTFT = time to **first** delta) and append to `outputText`; capture `usage`/`finishReason` when present; **`yield chunk` immediately** (zero added latency — SDK10/NFR1).
  4. On normal completion → `ship()`.
  5. On thrown error → if `err.name === 'AbortError'` or `opts.signal?.aborted`: `status = 'cancelled'`, `errorPayload = null`; else `status = 'error'`, `errorPayload = { code: 'provider_error', message: err.message ?? 'provider error' }`. Then `ship()` and **re-throw** so the caller's stream surfaces the failure.

  **`ship()`** (runs once, off the hot path):
  - `completedAtMs = Date.now()`; ISO `completedAt`; `latencyMs = completedAtMs - startedAtMs`.
  - Input preview source: last `user` message's content from `req.messages` (fallback: last message's content).

## Task 5: `BufferedHttpTransport` — buffered, non-blocking shipper (TDD)
**Implements:** SDK4 (in-memory bounded buffer, background flush on size/interval, never awaited by the response), SDK5 (exponential backoff + jitter up to `maxRetries`; drop on overflow/exhaustion with a local warning), SDK7 (config defaults). Implements the `LogSink` seam from Task 4. Tested against the **local mock ingestion HTTP server** from `test/fakes.ts` — no real network.
**Files:**
- Create: `packages/llm-sdk/src/transport/transport.ts` — `BufferedHttpTransport`, `TransportConfig`
- Edit: `packages/llm-sdk/src/logging/logger.ts` — append `withLoggingTransport` convenience factory
- Edit: `packages/llm-sdk/src/index.ts` — add value exports
- Test: `packages/llm-sdk/test/transport.test.ts`

**Design:**
- **Signatures / types:**
  ```ts
  // transport/transport.ts
  import type { InferenceLog } from '@ollive/shared';
  import type { LogSink } from '../logging/logger';

  export interface TransportConfig {
    ingestionUrl: string;
    apiKey: string;
    maxBufferSize?: number;    // default 500
    flushIntervalMs?: number;  // default 1000
    maxRetries?: number;       // default 3
    baseBackoffMs?: number;    // default 200
    autoFlushOnFull?: boolean; // default true
    fetchImpl?: typeof fetch;  // injectable for tests; defaults to globalThis.fetch
  }

  export class BufferedHttpTransport implements LogSink {
    constructor(config: TransportConfig);
    enqueue(log: InferenceLog): void; // non-blocking; drop-oldest on overflow
    flush(): Promise<void>;           // drains buffer; never throws
    bufferSize(): number;
    close(): Promise<void>;           // stop timer + final flush
  }

  // logging/logger.ts (append)
  export function withLoggingTransport(
    provider: LLMProvider,
    config: InferenceLoggerConfig,
  ): { provider: LLMProvider; transport: BufferedHttpTransport };
  ```
- **Algorithm:**
  - **Constructor** — apply defaults (`maxBufferSize=500`, `flushIntervalMs=1000`, `maxRetries=3`, `baseBackoffMs=200`, `autoFlushOnFull=true`); resolve `fetch = config.fetchImpl ?? globalThis.fetch` (bound to `globalThis`); start `setInterval(flush, flushIntervalMs)` and `.unref()` it so it never keeps the process alive.
  - **`enqueue(log)`** — no-op if closed. If `buffer.length >= maxBufferSize`, `shift()` the oldest and `console.warn('[llm-sdk] transport buffer overflow — dropping oldest log')`. Push the log. If `autoFlushOnFull` and now at capacity, fire-and-forget `void flush()`. **Never awaits** — returns instantly (NFR1).
  - **`flush()`** — re-entrancy guarded (a `flushing` boolean; return early if already flushing). While buffer non-empty, `shift()` one log and `await shipWithRetry(log)`. Never throws.
  - **`shipWithRetry(log)`** — loop `attempt` from `0` to `maxRetries` inclusive: POST to `ingestionUrl` with headers `content-type: application/json` + `authorization: Bearer <apiKey>`, body `JSON.stringify(log)`. If response ok or status `202` → delivered, return. If `4xx` and not `408`/`429` → non-retryable; `console.warn('[llm-sdk] dropping log: non-retryable status <n>')` and return. Else (`5xx`/`408`/`429`, or a thrown network error) → retryable: if this was the last attempt, `console.warn('[llm-sdk] dropping log after <maxRetries> retries')` (or after network failures) and return; otherwise sleep `baseBackoffMs * 2**attempt + random()*that` (exponential backoff + jitter) and retry.
  - **`close()`** — set closed, `clearInterval`, `await flush()`.
  - **`withLoggingTransport(provider, config)`** — construct a `BufferedHttpTransport` from `config` (passing `ingestionUrl`, `apiKey`, `maxBufferSize`, `flushIntervalMs`, `maxRetries`), then return `{ provider: withLogging(provider, config, transport), transport }` so the app can `await transport.close()` on graceful shutdown.
- **Patterns / decisions / edge cases:** Bounded buffer (drop-oldest on overflow) + background flush (size- or interval-triggered) keep the user response off the delivery path (SDK4/NFR1). Exponential backoff + jitter with a bounded retry budget (SDK5). Fire-and-forget enqueue (caller never awaits delivery). DI seam for `fetch` (`fetchImpl`) makes the transport testable; the `LogSink` interface is the same seam Task 4 ships to. `withLoggingTransport` is the factory wiring the decorator to the real transport. Edge cases: re-entrant `flush()` guarded; `4xx` (except `408`/`429`) is non-retryable (a bad payload won't improve); flush never throws — drops + warns instead, so chat is never affected; timer `.unref()` so it doesn't keep the process alive.

**Test cases (write first, TDD):** Build a valid `InferenceLog` via `inferenceLogSchema.parse({...})`. Spin up `makeMockIngestionServer` per test and `close()` in `afterEach`. To assert warnings, temporarily monkey-patch `console.warn` and restore it in `finally`.
- size-triggered flush + auth header — `maxBufferSize: 2`, large `flushIntervalMs`; enqueue 2 logs; `await flush()` → server received exactly 2 requests; first request's `authorization === 'Bearer secret-key'`; received body passes `inferenceLogSchema.parse`.
- interval flush — `maxBufferSize: 1000` (size never triggers), `flushIntervalMs: 20`; enqueue 1 log; wait ~80ms → server received exactly 1 request.
- retry on 5xx then succeed — server `failTimes: 2`, `baseBackoffMs: 5`, `maxRetries: 3`; enqueue 1 log; `await flush()` → server received 3 requests (2 failures + 1 success) for the same log.
- overflow drops oldest + warns — `maxBufferSize: 2`, `autoFlushOnFull: false`, unroutable URL (never flushed); enqueue 3 logs → `bufferSize() === 2` and a warning containing `'overflow'` was emitted.
- retry exhaustion drops + warns, no throw — server `failTimes: 100` (always 503), `maxRetries: 2`, `baseBackoffMs: 2`; enqueue 1 log; `await flush()` does **not** throw → server received 3 requests (1 initial + 2 retries) and a warning containing `'dropping'` was emitted.
